## Summary

Build the TV's in-memory song library domain ‚Äî `SongEntry` data model, `SongLibrary` aggregate index, `SongIndexer` (ParseResult ‚Üí SongEntry), and `SongDiscovery` (recursive .txt scan for tests). Extends the existing parser domain with `MALFORMED_BODY_FIELD` diagnostic code to enable clean validation error mapping. Uses the F01 fixture from `original_spec/fixtures/F01_song_discovery_validation_acceptance/` for acceptance tests.

See [research.md](research.md). No blocking unknowns. Key decisions:

## Implementation Notes

### Parser Extension (backward-compatible)

Add `MALFORMED_BODY_FIELD` to `DiagnosticCode` enum in `DiagnosticEntry.kt`.
Update `BodyParser.parse()` to emit `MALFORMED_BODY_FIELD` instead of `MALFORMED_NUMERIC_FIELD` for body-parse fatal errors (note/line-break token with non-parseable numeric field). No existing tests affected.

### SongEntry.invalidReasonCode Mapping

`SongIndexer.fromParseResult()` iterates `parseResult.parsedSong.diagnostics` for the first `DiagnosticSeverity.ERROR` and maps its `DiagnosticCode` to the stable spec string per the table in data-model.md.

### F01 Fixture Reuse

Copy F01 fixture to `app/src/test/resources/fixtures/library/song_discovery/` for Android classpath loading. The `expected.discovery.json` is read with `javaClass.classLoader.getResourceAsStream(...)` in acceptance tests.

### Sort Order

`DefaultSongLibrary.getSortedSongs()` uses `compareBy` on `Triple(artist.orEmpty().lowercase(), album.orEmpty().lowercase(), title.orEmpty().lowercase())`. Null album sorts before non-null albums.

### Gradle Tasks

Add to `app/build.gradle.kts` following the existing `scoring*` task pattern:

```kotlin
tasks.register<Test>("libraryUnitTest") {
    includes += "**/domain/library/**/*Test.class"
    excludes += "**/domain/library/**/*AcceptanceTest.class"
}
tasks.register<Test>("libraryAcceptanceTest") {
    includes += "**/domain/library/**/*AcceptanceTest.class"
}
tasks.register("libraryTest") {
    dependsOn("libraryUnitTest", "libraryAcceptanceTest")
}
```
