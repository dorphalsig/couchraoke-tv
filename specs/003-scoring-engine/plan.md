# Implementation Plan: Scoring Engine

**Branch**: `003-scoring-engine` | **Date**: 2026-03-15 | **Spec**: `/home/paavum/Couchraoke/tv/specs/003-scoring-engine/spec.md`
**Input**: Feature specification from `/specs/003-scoring-engine/spec.md`

## Summary

Build a pure Kotlin scoring engine inside the Android TV app that consumes finalized note windows from the beat timing engine and eligible pitch frames from phones, and computes per-note scores, line bonuses, and rounded display totals. The engine implements the normative USDX scoring formulas exactly: IEEE 754 hit-ratio note scoring, octave-normalized pitch matching with per-difficulty tolerance gates, line perfection with the forgiveness rule, line bonus distribution across non-empty lines, and asymmetric rounding for display integers. The implementation is a pure domain component with no new library dependencies.

## Technical Context

**Language/Version**: Kotlin 2.3.10 on Java 11
**Primary Dependencies**: Kotlin standard library, existing parser domain model (`NoteEvent`, `NoteType`, `ParsedSong`, `SongHeader`), existing beat timing domain model (`NoteTimingWindow`), existing Gradle/JUnit4 stack
**Storage**: N/A вЂ” pure in-memory domain computation; no persistence
**Testing**: JUnit 4 JVM tests plus fixture-driven acceptance tests under `app/src/test`
**Target Platform**: Android TV app; scoring itself is pure JVM/domain logic
**Project Type**: Mobile app feature with pure domain scoring component
**Performance Goals**: Deterministic per-note scoring and rounding with no floating-point divergence from normative formulas; suitable for synchronous invocation from a 100 Hz scoring polling loop without UI coupling
**Constraints**: Pure domain logic only; no Android framework types; JUnit4-only Android test tooling; no new library dependencies; IEEE 754 double-precision arithmetic throughout; `ScoreTotalInt` must never exceed 10000 for a perfect performance
**Scale/Scope**: One `PlayerScore` accumulator per player per track; one `ScoringConfig` per session; up to two tracks per song (duet songs have two independent accumulators)
